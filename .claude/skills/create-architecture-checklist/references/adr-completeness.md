# ADR 완성도 기준

## 필수 섹션 (전부 있어야 Accepted 가능)

| 섹션 | 내용 기준 |
|---|---|
| 상태 | `Proposed`·`Accepted`·`Deprecated`·`Superseded` 중 하나 |
| 컨텍스트 | 3문장 이상, 비즈니스·기술 배경 포함 |
| 결정 | 명확한 선택 1개 |
| 대안 검토 | 최소 3가지, 장단점 비교표 |
| 근거 | 트레이드오프 기반 선택 이유 |
| 결과 | 긍정적·부정적 영향 모두 기술 |

## 다이어그램 기준
- Mermaid 문법 유효성
- 컴포넌트·데이터 흐름 표현
- 시퀀스 또는 플로우차트 중 적합한 것 선택

## PRD 연계
- ADR이 PRD의 FR·NFR을 어떻게 충족하는지 명시
- NFR (성능·보안·확장성) 설계 반영 여부 확인
