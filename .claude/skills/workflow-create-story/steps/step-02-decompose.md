# Step 2: Epic → Story 분해

## 목적
INVEST 원칙에 따라 Epic을 독립적·구현 가능한 Story로 분해한다.

## INVEST 원칙
- **I**ndependent: 다른 Story에 독립적
- **N**egotiable: 협의 가능
- **V**aluable: 사용자 가치 제공
- **E**stimable: 추정 가능
- **S**mall: 1~3일 내 완료
- **T**estable: 테스트 가능

## 작업
1. FR별로 Epic을 정의하고 `docs/stories/<ticket>-EPIC-<n>.md`에 저장한다
2. 각 Epic을 Story로 분해한다
3. 각 Story가 INVEST 원칙을 충족하는지 확인한다
4. Story 간 의존성을 파악하고 순서를 정한다
5. 사용자에게 분해 결과를 보고하고 승인을 요청한다 (HITL L2)

## 완료 조건
- [ ] Epic 파일 `docs/stories/<ticket>-EPIC-<n>.md` 저장 완료
- [ ] 전체 Story 목록 작성
- [ ] INVEST 원칙 충족 확인
- [ ] 사람 승인 완료
